SQLite

Posted at 2019-07-23 Comments db 

参考：
http://www.runoob.com/sqlite/sqlite-tutorial.html

1. 安装

  • Linux
    自带
  • Windows
    需要下载sqlite-tools-win32-.zip 和 sqlite-dll-win32-.zip（或者64那个版本），然后将这两个解压到同一个目录中比如 C:\sqlite 然后将这个路径添加到PATH环境变量中

2. 命令

在命令行输入 sqlite3 进入SQLite提示符

[root@h104 ~]# sqlite3
SQLite version 3.6.20
Enter ".help" for instructions
Enter SQL statements terminated with a ";"
sqlite>

SQLite的命令格式： .（点） 命令名称
例如：


sqlite> .database   # 列出数据库的名称及其所依附的文件
seq  name             file    
---  ---------------  ------
0    main                            
sqlite> .exit       # 退出 SQLite 提示符

参考

  1. 官方文档
  2. Runoob

3. 基本操作

[root@h104 ~]# sqlite3 db1               #创建数据库db1   
SQLite version 3.6.20
Enter ".help" for instructions
Enter SQL statements terminated with a ";"
sqlite> .database
seq  name             file          
---  ---------------  
0    main             /root/db1                        
sqlite> create table t1(id int,name varchar(32));   # 创建表t1
sqlite> insert into t1 values(1,"zhang3");          # 插入一条数据  
sqlite> select * from t1;                           # 查询表t1  
1|zhang3
sqlite>

sqlite3 db1 创建一个名称为db1的数据库，会在当前目录下创建 db1文件

再创建一个数据库 db2

[root@h104 ~]# sqlite3 db2
SQLite version 3.6.20
Enter ".help" for instructions
Enter SQL statements terminated with a ";"
sqlite> create table t2(code int,desc varchar(32));
sqlite> insert into t2 values(66589,"it is a secret code");
sqlite> select * from t2;
66589|it is a secret code
sqlite>

再次选中db1库 ，linux与windows版本稍有不同。在windows中有open 和save命令，我使用的linux版本中 没有这两个命令

  • linux

    [root@h104 ~]# sqlite3 db1

  • windows

    E:\software\sqlite>sqlite3
SQLite version 3.29.0 2019-07-10 17:32:03
Enter ".help" for usage hints.
Connected to a transient in-memory database.
Use ".open FILENAME" to reopen on a persistent database.
sqlite> .open test1.db
sqlite> .database
main: E:\software\sqlite\test1.db
